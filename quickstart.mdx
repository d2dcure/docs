## Localhost Setup

### Prerequisites

- Node.js (v18 or higher)
- npm or yarn
- Git

### Installation

1. Navigate to the [GitHub Repository](https://github.com/d2dcure/d2d-cure)
2. Clone the Repository

   ```bash Terminal
   > git clone https://github.com/d2dcure/d2d-cure.git
   > cd d2d-cure
   ```
3. Install Dependencies

   ```bash Terminal
   > npm install
   ```
4. Set up enviornment variables
   - Create a `.env`  and `.env.local` file in the root directory with the following variables:

   ```bash .env
   DATABASE_URL_USERS="key_here"
   DATABASE_URL_PROTEINS="key_here"
   DATABASE_URL_ENZYMES="key_here"
   DATABASE_URL_PUBLICATIONS="key_here"
   FIREBASE_PRIVATE_KEY="key_here"
   ```

   ```bash .env.local
   AWS_ACCESS_KEY_ID=key_here
   AWS_SECRET_ACCESS_KEY=key_here
   ```

   <Note>
     Note: the secret keys are not publicly available: please contact support to obtain the actual values for these keys. For simple frontend changes, secrets keys are not necessary. 
   </Note>
5. Start the development server

   ```bash Terminal
   > npm run dev
   ```

## Troubleshooting

### Common Issues

1. **Database Connection Errors**
   - Check DATABASE_URL in .env
   - Verify database server status
2. **Authentication Issues**
   - Verify Firebase configuration
   - Check user permissions
3. **File Upload Problems**
   - Confirm AWS credentials
   - Check file size limits

## Viewing the Database Structure

There are several tools that exist to be able to view the MariaDB database structure (database names, tables, columns, values, etc). One of the most popular and user-friendly options is **MySQL Workbench**.

### Setup

1. Download MySQL Workbench from the official website (https://dev.mysql.com/downloads/workbench/)
2. Install and launch the application

### Connecting to the Database

1. Click the "\+" icon to create a new connection
2. Configure connection settings:

- Connection Name: D2D-Cure-DB
- Hostname: [Your database host]
- Port: 3306
- Username: [Your database username]
- Password: [Your database password]

3. Test the connection and click "OK" to save

<Note>
  Note: please contact support to recieve the actual values for the connection settings
</Note>

### Using MySQL Workbench

- **Browse Structure**: Expand the connection in the sidebar to view databases, tables, and columns
- **Execute Queries**: Open a new query tab to run SQL commands
- **View Data**: Right-click on tables and select "Select Rows" to see data
